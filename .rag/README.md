# RAG Workspace

This directory separates durable RAG curation files from generated vector-store
artifacts.

- `manifests/*.sections.json` are curated section-boundary manifests. Draft them
  deterministically with `uv run python scripts/rag.py sections --write`, then
  review and edit them when boundaries are weak.
- `reviews/*.section-review.md` are LLM-facing review packets generated by
  `uv run python scripts/rag.py review-sections --write`. They give an agent
  enough line/page context to check and repair section manifests.
- `chroma/` and `index-manifest.json` are generated index artifacts and remain
  ignored.

Indexing should read reviewed manifests when they exist. Avoid relying on fresh
formatting heuristics for important retrieval work.
